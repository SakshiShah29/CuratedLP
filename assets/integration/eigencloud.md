# EigenCloud Integration Guide

## Setup for CuratedLP — EigenCompute TEE Deployment

---

## Overview

EigenCompute deploys Docker containers to Trusted Execution Environments (Intel TDX).
No API key needed. Auth is wallet-based (private key + ETH for gas).

CuratedLP deploys the Venice AI inference pipeline (sentiment + analysis) inside
an EigenCompute TEE. The TEE attestation proves both Venice calls ran unmodified.

---

## Prerequisites

- **Docker** — to package and publish images ([Download](https://www.docker.com/))
- **Testnet or Mainnet ETH** — for deployment transactions
- **VENICE_API_KEY** — injected as env var into the TEE at runtime

## Step 2: Authenticate

EigenCompute uses wallet-based auth (private key), not API keys.

```bash

# Option B: Generate a new key
ecloud auth generate --store
```

Check your wallet address:

```bash
ecloud auth whoami
```

Output:


## Step 6: Deploy to EigenCompute TEE

```bash
cd agent

# Log in to Docker registry (required to push image)
docker login

# Deploy
ecloud compute app deploy
```

When prompted, select **"Build and deploy from Dockerfile"**.

The CLI will:
1. Build Docker image targeting `linux/amd64`
2. Push to Docker registry
3. Deploy to a TEE instance
4. Return app ID + instance IP

### View your app

```bash
ecloud compute app info
```

### Set environment variables

The VENICE_API_KEY needs to be configured in the `.env` file before deploying.
EigenCompute injects env vars into the TEE encrypted — they're only decrypted
inside the enclave.

The `MNEMONIC` env var is auto-generated by KMS at runtime. Any value in `.env`
is overwritten with the app's unique, persistent KMS-generated mnemonic.

Variables with `_PUBLIC` suffix are visible to users for transparency.
All other variables remain encrypted within the TEE.

## Step 7: Configure eigencompute.ts to Use the TEE

After deploying, set the endpoint in `agent/.env`:

```bash
EIGENCOMPUTE_MODE=tee
EIGENCOMPUTE_ENDPOINT=https://<your-instance-ip-or-domain>
```

Test it:

```bash
npx tsx src/tools/eigencompute.ts \
  --pool '{"currentTick":-201840,"tickLower":-202200,"tickUpper":-201400,"totalLiquidity":"50000000","currentFee":3000,"idleToken0":"0.02","idleToken1":"0","accruedPerformanceFee":"0.001"}' \
```

## Architecture

```
  eigencompute.ts (agent tool)
       │
       │ POST /inference with pool state + uniswap data
       ▼
  ┌──────────────────────────────────────────────┐
  │  EigenCompute TEE (Intel TDX)                │
  │                                              │
  │  Docker: curatedlp/venice-analyzer           │
  │    │                                         │
  │    │ VENICE_API_KEY injected via TEE KMS     │
  │    │                                         │
  │    ▼                                         │
  │  Venice Call #1 (web search ON)              │
  │    → sentiment: bullish/bearish/neutral      │
  │    → signals: governance, whales, social     │
  │    │                                         │
  │    ▼                                         │
  │  Venice Call #2 (web search OFF)             │
  │    → input: pool + uniswap + sentiment       │
  │    → output: tick range + fee + confidence    │
  │    │                                         │
  │    ▼                                         │
  │  sentiment + recommendation JSON             │
  │    + content integrity hash                  │
  │                                              │
  │  TEE attestation proves:                     │
  │    - Code ran unmodified (Docker digest)      │
  │    - Venice web search was executed           │
  │    - Data was not tampered with               │
  │    - Recommendation was not altered           │
  └──────────────────────────────────────────────┘
       │
       │ sentiment + recommendation + attestation
       ▼
  OpenClaw Agent → olas-analyze → execute-rebalance
```

## Dockerfile Requirements

EigenCompute TEE requires:

```dockerfile
FROM --platform=linux/amd64 node:20-slim  # Must be amd64
USER root                                  # Must run as root in TEE
EXPOSE 3000                                # Must expose port
# App must bind to 0.0.0.0 (not localhost)
```

Our Dockerfile (`agent/Dockerfile`) already meets all these requirements.


## Troubleshooting

| Problem | Solution |
|---|---|
| Docker build fails | Ensure `--platform linux/amd64` flag |
| Deployment tx fails | Check ETH balance with `ecloud auth whoami` |
| Image push fails | Run `docker login` first |
| TEE app won't start | Check `ecloud compute app logs`, ensure `0.0.0.0` binding |
| App port not accessible | Ensure `EXPOSE 3000` in Dockerfile |
| Missing env vars in TEE | Add to `.env` before deploying, redeploy with `ecloud compute app upgrade` |
| eigencompute.ts falls back to Venice | TEE unavailable — check endpoint URL and app status |
